# 00 架构与技术栈

本项目是对 **da-ai.cc「丢抖AI 电商工作台」** 的像素级（1:1）静态克隆，使用现代 Next.js 技术栈重建。所有数据均为内置的 mock 静态数据，无后端接口，聚焦于 UI 还原与前端交互。

## 技术栈

| 层 | 选型 | 说明 |
| --- | --- | --- |
| 框架 | **Next.js 16**（App Router, Turbopack） | 全部页面位于 `src/app/`，以文件路由组织 |
| UI 库 | **React 19** + TypeScript（strict） | 禁用 `any`，命名导出，组件 PascalCase |
| 样式 | **Tailwind CSS v4** | 设计令牌以 CSS 变量 + `@theme inline` 定义于 `globals.css` |
| 组件基座 | **shadcn/ui**（Radix / `@base-ui/react`） | `src/components/ui/`，`cn()` 合并类名 |
| 图标 | **lucide-react** + 自绘 SVG | 画布模块自绘图标见 `src/components/daai/canvas/icons.tsx` |
| 画布 | **@xyflow/react** | 高级画布编辑器的节点/连线基础 |
| 部署 | Vercel / 容器（`output: "standalone"`） | 见 `Dockerfile`、`docker-compose.yml` |

## 目录结构

```
src/
  app/
    layout.tsx                     # 根布局：<html lang="zh-CN">、全站 metadata/SEO
    page.tsx                       # 根路由 → 重定向到 /agent/ecommerce/home
    globals.css                    # 设计令牌（颜色/圆角/字体）+ 基础样式
    login/                         # 登录页（独立 layout，不带工作台外壳）
    agent/ecommerce/
      layout.tsx                   # 工作台外壳（canvas/[id] 全屏编辑器除外）
      home/  copywriting/  canvas/  remix/  video/
      image/  assets/  settings/  admin/  credits/  library/
  components/
    Sidebar.tsx  Header.tsx  WorkspaceShell.tsx   # 布局三件套
    daai/HomeComposer.tsx  QuickEntry.tsx  DiscoverFeed.tsx  Footer.tsx  # 首页四大区块
    daai/canvas/                                  # 高级画布编辑器全套组件
    ui/                                           # shadcn 基础组件
  lib/
    utils.ts        # cn() 类名合并工具
    mock-data.ts    # 侧栏导航、提取 tab、案例展示等 mock
    daai-data.ts / assets-data.ts / canvas-data.ts  # 各模块 mock 数据
  types/index.ts    # 全局 TS 类型
public/             # 图片 / 品牌资源 / SEO
docs/               # 逆向研究产物（设计参考图、组件规格）
doc/                # 👈 本套「按菜单功能」的中文技术文档
```

## 路由与导航

- 根路由 `/` 通过 `redirect()` 跳转到 `/agent/ecommerce/home`，与侧栏「首页」入口保持一致。
- 侧栏主菜单由 `src/lib/mock-data.ts` 的 `SIDEBAR_NAV` 驱动，底部区（积分 / 订阅 / 退出）由 `SIDEBAR_BOTTOM` 驱动。
- 工作台所有页面共享 `WorkspaceShell`（侧栏 + 顶栏 + 滚动主区）；**唯一例外**是全屏画布编辑器 `/agent/ecommerce/canvas/[id]`，它在 `agent/ecommerce/layout.tsx` 中被判定跳过外壳、自带全屏 chrome。

## 设计令牌（`globals.css`）

均为逐值从参照站 `getComputedStyle` 抄录，保证像素级一致：

| 令牌 | 值 | 用途 |
| --- | --- | --- |
| `--primary` / `--ring` | `#0471fe` | 主色（品牌蓝） |
| `--background` | `#f4f4f5` | 页面底色（zinc-100） |
| `--workspace-bg` | `#f6f8fb` | 工作台主滚动区底色 |
| `--foreground` | `#18181b` | 主文本色 |
| `--card` | `#ffffff` | 卡片面 |
| `--radius-sm…2xl` | 8/10/14/18/24px | 圆角梯度 |
| 字体栈 | `ui-sans-serif, system-ui, …, "PingFang SC", "Microsoft YaHei"` | 与参照站一致 |
