# 12 公共组件与布局

本篇覆盖跨菜单复用的布局外壳与公共组件，它们不隶属于单个菜单，而是支撑整个工作台。

## 根布局与入口

| 文件 | 职责 |
| --- | --- |
| `src/app/layout.tsx` | 根布局，`<html lang="zh-CN">`，注入全站 `metadata`（标题 / 描述 / favicon / apple-icon / webmanifest） |
| `src/app/page.tsx` | 根路由，`redirect("/agent/ecommerce/home")`，把 `/` 统一导向首页 |
| `src/app/globals.css` | 设计令牌（CSS 变量）+ Tailwind v4 `@theme inline` 映射 + 基础样式，详见 [`00-架构与技术栈.md`](./00-架构与技术栈.md) |

## 工作台外壳 WorkspaceShell

> `src/components/WorkspaceShell.tsx`

工作台三段式布局的组合根：左 `Sidebar` + 右侧 `Header` + 可滚动 `main`。

- 用 `useState` 维护 `sidebarOpen`（移动端抽屉开合），下发给 `Sidebar`（`open` / `onClose`）与 `Header`（`onToggleSidebar`）。
- 主区 `max-w-shell` 居中，`overflow-y-auto` 独立滚动，`bg-workspace` 底色。
- 在 `agent/ecommerce/layout.tsx` 中被套用于**除全屏画布编辑器外**的所有工作台页面。

## 侧边栏 Sidebar

> `src/components/Sidebar.tsx`

- 数据驱动：主菜单来自 `SIDEBAR_NAV`，底部区来自 `SIDEBAR_BOTTOM`（均在 `src/lib/mock-data.ts`）。
- 桌面端为 76px 窄条固定栏；移动端为 `min(82vw,280px)` 抽屉，配半透明遮罩 + `translate-x` 过渡。
- 用 `usePathname()` 高亮当前路由（`aria-current="page"`），激活项加主色底与描边阴影。
- 底部区三种 `variant`（`credits` / `billing` / `logout`）各有独立配色，通过 `BOTTOM_STYLES` 映射；有 `href` 渲染为 `Link`，否则为 `button`。
- 全站统一的悬浮微动效（`hover:-translate-y-0.5 hover:scale-[1.03]`）并遵守 `motion-reduce`。

## 顶栏 Header

> `src/components/Header.tsx`

- 左侧：侧栏切换按钮（`PanelLeft`，移动端展开抽屉）+ 站点标题「丢抖AI素材工作台」与副标题面包屑文案。
- 右侧：`PillLink` 快捷入口（「文案/分镜库」→ `/library`、「设置」→ `/settings`）。
- `PillLink` 为本文件内部小组件，统一胶囊按钮样式。

## 首页展示组件

这四个组件构成「首页」主体，详见 [`01-首页-home.md`](./01-首页-home.md)。

| 组件 | 文件 | 职责 |
| --- | --- | --- |
| `HomeComposer` | `src/components/daai/HomeComposer.tsx` | 文案提取多 tab（链接/视频/图片/录音/音频）+ 提取结果 + 「快速成片」下拉（数字人成片/分镜与片段生成/智能成片/高级画布），含用时/完成提示等模拟态 |
| `DiscoverFeed` | `src/components/daai/DiscoverFeed.tsx` | 成品展示：分段 tab（全部/视频/图片）过滤 `DISCOVER_ITEMS`，CSS columns 瀑布流，卡片为 da-ai.cc 覆盖式布局（类型图标 + 标题 + 爱心点赞） | 
| `Footer` | `src/components/daai/Footer.tsx` | 首页底部信息栏 |

## 全局类型 `src/types/index.ts`

集中定义跨模块 TS 类型：`SidebarNavItem`、`SidebarBottomItem`、`CreationMode(Item)`、`ExtractTab(Key)`、`ShowcaseTab(Key)`、`ShowcaseItem`、`MediaAspect`、`LucideIcon` 等，供各页面与 mock 数据共享，保证 strict 模式下的类型一致。

## Mock 数据层

| 文件 | 内容 |
| --- | --- |
| `src/lib/mock-data.ts` | 侧栏导航 / 底部区、提取 tab、创作模式、创作广场案例等共享 mock |
| `src/lib/daai-data.ts` | 部分模块页引用的业务 mock |
| `src/lib/assets-data.ts` | 资产库数据 |
| `src/lib/canvas-data.ts` | 高级画布节点/模板数据 |
| `src/lib/utils.ts` | `cn()` —— `clsx` + `tailwind-merge` 类名合并 |

> 注：多数菜单页（remix / video / image / settings / admin 等）为自包含静态原型，数据以**页面内联常量**为主，未统一收敛到 `lib/`，这一现状在各模块文档中已如实标注。
